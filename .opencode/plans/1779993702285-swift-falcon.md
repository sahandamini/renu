# Railway Build Failure Plan

## Diagnosis

Railway failed during dependency installation, before the app build. The project uses `packageManager: "pnpm@11.4.0"`, but `package.json` still stores pnpm configuration in the deprecated top-level `"pnpm"` field. pnpm 11 ignores that field, so `onlyBuiltDependencies`, `overrides`, and `peerDependencyRules` are not applied.

The log also shows `prepare` running `vp config` during install. `vp config` tries to configure Git hooks and prints `.git can't be found` because Railway's build environment does not expose a normal repository `.git` directory. Vite+ can skip that hook setup with `VITE_GIT_HOOKS=0` or `HUSKY=0`.

The fatal error is `ERR_PNPM_IGNORED_BUILDS`: pnpm detected dependency build scripts for `core-js@3.49.0` and `protobufjs@7.6.1` that were not approved under pnpm 11's active config.

## Recommended Changes

1. Add `pnpm-workspace.yaml` at the repository root.
2. Move the existing pnpm settings from `package.json` into `pnpm-workspace.yaml`.
3. Add `core-js` and `protobufjs` to `onlyBuiltDependencies`, alongside the existing `esbuild` and `lightningcss` approvals.
4. Remove the ignored top-level `"pnpm"` block from `package.json`.
5. Add a Railway/Nixpacks environment config so install-time hook setup is skipped in deploy builds.

## Files To Modify

- `package.json`
- `pnpm-workspace.yaml`
- `nixpacks.toml`

## Intended Config

`pnpm-workspace.yaml` should contain:

```yaml
packages:
  - "."

onlyBuiltDependencies:
  - core-js
  - protobufjs
  - esbuild
  - lightningcss

overrides:
  vite: npm:@voidzero-dev/vite-plus-core@latest
  vitest: npm:@voidzero-dev/vite-plus-test@latest

peerDependencyRules:
  allowAny:
    - vite
    - vitest
  allowedVersions:
    vite: "*"
    vitest: "*"
```

`nixpacks.toml` should contain:

```toml
[variables]
VITE_GIT_HOOKS = "0"
HUSKY = "0"
```

## Verification

Run locally:

```sh
VITE_GIT_HOOKS=0 HUSKY=0 pnpm install --frozen-lockfile --prefer-offline
vp check
vp test
pnpm run build
```

Then redeploy on Railway and confirm the install no longer emits the ignored `package.json` pnpm settings warning or `ERR_PNPM_IGNORED_BUILDS`.
