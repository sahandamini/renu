# ReNu Homepage Redesign Plan

## Goal

Overhaul the MVP into a more elevated, serene, boutique medical spa site while keeping the current TanStack Start/React structure. Focus this pass on the homepage, header, footer, global styling, and localized brand imagery. Use one polished default visual mode rather than a visible light/dark theme experience.

## User Direction

- Visual feel: elevated, serene, boutique, calm, high-end, confidence-inspiring.
- Brand palette: soft neutrals, deep charcoal/black accents, warm gold tones.
- Positioning: private wellness destination, not a traditional clinic.
- Messaging priorities: trust, comfort, medical credibility, personalized care, luxurious but approachable experience.
- Scope: homepage first, including shared header/footer/global styling.
- Assets: localize current remote Squarespace imagery instead of relying on remote CDN URLs.
- Theme: one refined default mode.

## Critical Files

- `src/routes/index.tsx`: main homepage; currently contains all homepage content, image URLs, service data, and section markup.
- `src/styles.css`: Tailwind v4 CSS-first theme, shadcn-compatible variables, fonts, global background, and reusable visual classes.
- `src/components/Header.tsx`: sticky navigation and remote logo usage.
- `src/components/Footer.tsx`: footer content and dark visual treatment.
- `src/routes/__root.tsx`: global metadata, stylesheet links, theme initialization script, remote social image, and Squarespace preconnects.
- `public/manifest.json`: still has starter app values; update if touching public brand assets.
- `public/assets/brand/`: create for localized logo and homepage imagery.

## Implementation Steps

1. Localize Brand Assets
   - Create `public/assets/brand/`.
   - Download the current Squarespace logo and homepage images referenced in `src/routes/index.tsx`, `src/components/Header.tsx`, and `src/routes/__root.tsx`.
   - Save descriptive filenames, for example:
     - `renu-logo.png`
     - `hero-provider.jpg`
     - `consultation.jpg`
     - `services-treatment.jpg`
     - `membership-room.jpg`
     - `shop-wellness.png`
     - `exterior.jpg`
   - Replace remote URLs with root-relative paths such as `/assets/brand/hero-provider.jpg`.
   - Use a strong homepage image as the local OG/Twitter image in `src/routes/__root.tsx`.

2. Retune Global Design System
   - Update `src/styles.css` variables for a single warm default palette:
     - warm ivory/linen background
     - deep charcoal/espresso foreground
     - muted champagne/gold accent
     - soft sand secondary surfaces
     - understated borders and shadows
   - Keep shadcn-compatible variables so future shadcn components inherit the brand correctly.
   - Remove or stop depending on the active `.dark` branch since the site should have one default visual mode.
   - Replace the glassy/card-heavy feel with more refined editorial surfaces: fewer boxed sections, softer dividers, more whitespace, better rhythm.
   - Keep reusable classes minimal and practical, such as `page-wrap`, `display-title`, `island-kicker` or renamed equivalent, `island-shell` or renamed equivalent, `feature-card`, `nav-link`, and `site-footer`.
   - Prefer existing typography unless visual testing shows it is still a mismatch: Cormorant Garamond can work for refined display headings; Montserrat may need lighter usage and more letterspacing restraint.

3. Rebuild Homepage Markup In Place
   - Modify `src/routes/index.tsx` rather than introducing unnecessary new components.
   - Keep current business links and contact details:
     - booking: `https://renunola.janeapp.com/`
     - shop: `https://shop.renunola.instapaytient.com/`
     - membership: `https://shop.renunola.instapaytient.com/shop/renu-me-membership`
     - Instagram: `https://www.instagram.com/renunola`
     - address, hours, phone, and email as currently shown.
   - Hero direction:
     - More editorial/private-lounge composition.
     - Strong but calm headline about refined aesthetics, restorative wellness, and personalized medical care.
     - Primary CTA: `Book Online`.
     - Secondary CTA: `Explore Services`.
     - Local imagery layered with restraint, not excessive cards.
   - Trust/credibility direction:
     - Replace chunky highlight boxes with an elegant proof strip or quiet three-column trust band.
     - Emphasize licensed providers, personalized plans, and New Orleans boutique wellness.
   - About direction:
     - Position ReNu as a serene private wellness destination with clinical expertise.
     - Pair concise copy with localized exterior/interior imagery.
   - Services direction:
     - Keep existing service list unless copy refinements are needed.
     - Present services as refined, readable cards or editorial rows with subtle numbering and restrained hover effects.
     - Avoid the current generic boxed/card grid feeling.
   - Membership direction:
     - Keep benefits and membership URL.
     - Present as a curated concierge wellness plan, not a sale box.
   - Shop/Instagram direction:
     - Keep both as supporting conversion paths.
     - Reduce visual weight compared with booking/services.
   - Contact direction:
     - Make the final CTA calm and confidence-building.
     - Keep contact cards readable on mobile without feeling like a dashboard.

4. Refresh Header
   - Update `src/components/Header.tsx` to use `/assets/brand/renu-logo.png`.
   - Keep anchor links and external booking/shop/Instagram links.
   - Make the header slimmer, more boutique, and less visually busy.
   - Keep `Book Online` prominent.
   - Verify mobile wrapping. Add a mobile menu only if the polished header cannot fit well without one.

5. Refresh Footer
   - Update `src/components/Footer.tsx` to match the new deep-charcoal, ivory, and gold visual language.
   - Keep contact information and important links concise.
   - Optionally include the local logo or a refined text lockup.

6. Remove Single-Mode Theme Leftovers
   - In `src/routes/__root.tsx`, remove `THEME_INIT_SCRIPT` and its inline `<script>`.
   - Remove `suppressHydrationWarning` if it is only needed for theme class hydration.
   - Remove Squarespace image preconnects after assets are localized.
   - Update `socialImage` to a local public asset path.
   - Leave `ThemeToggle.tsx` unused unless later requested; no visible theme toggle should be added.

7. Update Public Metadata Where Appropriate
   - Update `public/manifest.json` starter values to ReNu branding if included in the pass:
     - `short_name`: `ReNu NOLA`
     - `name`: `ReNu Health and Wellness`
     - warm ivory background and charcoal/gold theme color.

## Constraints

- Do not change frameworks or routing architecture.
- Do not edit `src/routeTree.gen.ts` directly.
- Do not add shadcn components unless there is a concrete need; current design can be achieved with existing Tailwind/shadcn-compatible CSS variables.
- Keep changes homepage-first; do not spend time fully redesigning `/about` or `/demo/posthog` in this pass.
- Keep implementation small and direct. Avoid over-componentizing unless repeated markup becomes hard to maintain.
- Preserve accessibility basics: semantic headings, descriptive image alt text, visible focus states through existing ring variables, readable contrast, and sensible mobile order.
