# ReNu Health and Wellness Landing Page Plan

## Goal

Create a polished first-pass one-page landing page for ReNu Health and Wellness using the current Vite+ / TanStack Start / Tailwind v4 project. The initial version will use anchored sections, remote Squarespace logo/photo URLs, and external links for booking, shop, membership purchase, and Instagram. Separate pages can be added iteratively after the landing page establishes the brand, theme, and content structure.

## Confirmed Decisions

- Build a one-page landing page first, not separate routes yet.
- Use existing remote Squarespace image/logo URLs initially.
- Keep implementation minimal: no new dependencies unless a concrete need appears.
- Preserve the existing app structure and Vite+ workflow.

## Content And Links

- Brand: ReNu Health and Wellness / ReNu Nola.
- Main nav/sections: About, Services, Ultimate Membership, Shop, Contact, Instagram.
- Primary CTA: Book Online, `https://renunola.janeapp.com/`.
- Shop: `https://shop.renunola.instapaytient.com/`.

## Recommended Design Direction

- Visual tone: premium medical spa, warm clinical trust, calm beauty/wellness.
- Use the existing Fraunces + Manrope typography as a strong editorial/premium pairing.
- Reuse the current teal, green, ivory, and glass-panel styling, but remove all starter-template language.
- Use real clinic photography in asymmetrical image cards to avoid a generic SaaS-style layout.
- Keep CTAs direct and appointment-focused: `Book Online`, `Explore Services`, `Join the Membership`, `Shop Wellness`, `Follow on Instagram`.

## Implementation Steps

1. Update `src/routes/index.tsx`.
   - Replace TanStack starter content with a semantic one-page landing page.
   - Define constants for external links and image URLs near the top of the file.
   - Define `services`, `contactItems`, and optional `trustHighlights` arrays in the same file for now.
   - Build sections with stable IDs: `about`, `services`, `membership`, `shop`, `instagram`, `contact`.
   - Use existing classes: `page-wrap`, `island-shell`, `feature-card`, `display-title`, `island-kicker`, `rise-in`.
   - Structure sections as hero, trust strip, about, services grid, membership feature, shop/social CTA, contact.
   - Use one `h1`; use `h2` for section headings; add `aria-labelledby` where useful.
   - Use lazy loading for below-the-fold images and stable aspect-ratio wrappers to reduce layout shift.

2. Update `src/components/Header.tsx`.
   - Replace starter brand with ReNu logo/name.
   - Replace TanStack nav with anchor links: About, Services, Membership, Shop, Contact, Instagram.
   - Add a prominent `Book Online` CTA linking to Jane App.
   - Keep the sticky translucent header and responsive wrapping behavior.
   - Keep `ThemeToggle` for the first pass unless the brand should become light-only later.

3. Update `src/components/Footer.tsx`.
   - Replace placeholder copyright and TanStack social links.
   - Add ReNu copyright, address, phone, email, hours, and quick links.
   - Include Book Online, Shop, Contact, and Instagram links.

4. Update `src/routes/__root.tsx`.
   - Change metadata title to `ReNu Health and Wellness | Medical Spa in New Orleans`.
   - Add a description covering aesthetics and wellness services.
   - Add Open Graph/Twitter metadata using a ReNu image URL.
   - Add preconnect links for Squarespace image hosts if images are loaded remotely.

5. Update `src/styles.css` minimally.
   - Add `html { scroll-behavior: smooth; }`.
   - Add a `prefers-reduced-motion` rule to disable smooth scrolling and `rise-in` animation.
   - Add tiny reusable CTA/link helper classes only if repeated Tailwind class strings become noisy.
   - Avoid broad theme rewrites until the first landing page is visible.

## Critical Files

- `src/routes/index.tsx`: primary landing page content and layout.
- `src/components/Header.tsx`: brand, nav, and primary booking CTA.
- `src/components/Footer.tsx`: contact details and secondary links.
- `src/routes/__root.tsx`: title, description, social metadata, resource hints.
- `src/styles.css`: global tokens, page shell utilities, motion and scroll behavior.

## Verification

1. Run `vp install` if dependencies may be stale.
2. Run `vp check` to format, lint, and type-check.
3. Run `vp test` to verify the project baseline.
4. Run `vp build` to verify production build output.
5. Run `vp dev --port 3000` and manually inspect desktop and mobile widths.
6. Check that all anchors scroll correctly: `#about`, `#services`, `#membership`, `#shop`, `#instagram`, `#contact`.
7. Check external links: Book Online, Shop, Membership purchase, Instagram, phone, and email.
8. Confirm the page remains usable with keyboard focus and reduced-motion preferences.
